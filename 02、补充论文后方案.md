一、 系统总体架构
系统分为四层：感知层、决策算法层、交互层、云平台通信层。
1. 感知层 (Data Acquisition)
• 运动感知： IMU（加速度计+陀螺仪）采集头部姿态、震动。
• 环境感知： 温湿度传感器、光敏电阻（自动灯光）、GPS（位置与速度）。
• 后方感知： 超声波雷达或毫米波雷达（盲点监测）。
• 生理感知： 心率血氧传感器（贴合额头或耳垂）。
感知层优化：
• 冲击感应增强：建议引入**压电传感器（Piezoelectric Sensor）**作为IMU的低成本辅助，专门用于检测头部的直接机械创伤强度（Trauma Degree），其成本极低且响应迅速。
• 光感自动补光：利用简单的光敏电阻实现头盔灯光的自动调节，提高隧道或夜间骑行的安全性。

---

2. 决策算法层 (Edge Computing)
• 跌倒判定逻辑： 双阈值判定法（冲击+静止）。
• 姿态评估： 俯仰角分析（风阻姿态）与频域分析（疲劳检测）。
• 数据平滑： 卡尔曼滤波（针对 GPS 轨迹和 IMU 数据）。
决策算法层优化：
• 跌倒判定多阶段化：采用基于阈值的方法（TBM），分为“失重/自由落体”、“冲击（Impact）”和“伤后静止（Post-fall Inactivity）”三个连续阶段判定，以减少骑行颠簸造成的误报。

3. 交互与控制层 (HMI)
• 视觉： 透明 OLED/棱镜（HUD）、WS2812 RGB 灯带（转向灯/预警灯）。
• 触觉/听觉： 左右振动马达（盲区提醒）、蜂鸣器（报警）。

4. 通信层 (Connectivity)
• 4G 全网通： MQTT 协议与云端/亲友端交互。
• V2X 同步： 订阅/发布车队主题，实现组队同步。

1. 碰撞识别与跌倒确认逻辑
• 增加后处理确认：在判定跌倒发生后，系统应进入4秒以上的静止监测期。
• 人机交互二次确认：通过HUD或语音询问“你受伤了吗？”，若用户在30秒内未点击“我很好”，则自动触发救援。这能有效解决阈值法无法处理复杂个体差异的问题。
• 滤波优化：使用卡尔曼滤波（Kalman Filter）消除IMU的累积漂移和高频噪声，特别是在计算俯仰角和侧倾角时。

2. 后方盲点监测系统 (Active Defense)
• 实现： 后置超声波传感器。
• 逻辑：
    ◦ 距离 ( d < 2m )：两侧灯带红色闪烁。
    ◦ 距离 ( d < 1m ) 且相对速度为正：触发对应侧马达振动，提醒骑行者。
• 过滤算法： 针对路边栏杆等静态物体，通过 GPS 速度补偿进行差值过滤，仅对快速靠近的动态物体预警。
• 区域定义：参考汽车标准，将监测范围设定为侧后方约45°夹角、距离12米以内，这能覆盖最危险的换道盲区。
• 低成本方案对比：虽然超声波方案最廉价，但其监测距离较短。若未来增加摄像头，可采用检测窗口（Detection Window）算法，仅处理特定图像区域以节省计算资源，确保在低成本嵌入式平台上实现实时监控。

4. V2X 车队协同同步 (Group Synergy)
• 逻辑： 采用“领骑-跟骑”模型。
• 转向同步： 领骑者按下头盔上的转向按钮，STM32 通过 4G 发送指令至 MQTT Topic Team_Alpha_Action。所有跟骑头盔订阅此 Topic，并在 200ms 内同步闪烁对应的转向灯。
• 协议选型：推荐使用C-V2X (LTE-V2X) 模式，其在密集环境下比传统的DSRC更稳定，且通过现有的4G/5G基础设施即可实现，能提供Predictable Delay（延时低于100ms）。

五、 实现路径建议
1. 第一步（MVP版本）： 在 UniKnect 开发板上跑通 IMU 跌倒算法 + 4G GPS 上传。这是整个项目的基础核心。
2. 第二步（交互增强）： 实现 WS2812 自动灯光控制和超声波避障，将预警反馈通过灯光和振动表现出来。
3. 第三步（智能化提升）： 加入 MicroPython 下的简单卡尔曼滤波，优化 GPS 轨迹，并尝试接入心率模块。
4. 第四步（V2X）： 编写简单的 Python 后台，实现两台开发板之间的 MQTT 联动。
• 误报问题 (False Positives)：单纯的阈值法（TBM）在跳跃或剧烈运动中易误报。建议通过个人化阈值调整（根据用户的年龄、体重、历史运动习惯动态微调阈值）来优化。
功耗挑战：
• 动态控制频率：在GPS检测到速度降为零或长时间无头部运动时，降低IMU采样率和4G模组心跳包频率。
• 物理安装：将电子系统安装在头盔后部（Backside）以保持重心平衡。
---
六、 潜在挑战与解决方案
• 挑战 1： 4G 模组功耗较高。
    ◦ 方案： 采用动态心跳频率，当 GPS 速度低于 1km/h 超过 5 分钟，进入低功耗待机模式。
• 挑战 2： 震动干扰跌倒识别。
    ◦ 方案： 引入频域特征分析，滤除骑行时高频震动带来的加速度伪峰。
• 挑战 3： 户外强光导致 OLED 看不清。
    ◦ 方案： HUD 设计需考虑半透半反棱镜结构，或选用高亮度 LED 阵列作为指示。
• 第一步（MVP+确认机制）：跑通IMU跌倒算法，并加入30秒伤后用户二次确认的交互逻辑。
• 第二步（传感器融合）：加入压电传感器（机械冲击）与IMU（运动状态）的融合判定，提升判定健壮性。
• 第三步（HMI精简化）：实现HUD的图形指示，优先做“车辆靠近警示”，采用红色LED阵列或透明OLED的强警告。
• 第四步（云端监控）：利用现有的物联网平台（如ThingsBoard或Firebase）实现轨迹实时分享给亲友
